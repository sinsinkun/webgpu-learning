WebGPU Canvas


Details:

the canvas is drawn on using the new WebGPU API, which will be overtaking the old WebGL API that is currently used for most web related rendering tasks. WebGPU also brings with it a new shader language, which should be easier to work with than the old GLSL standard. Beyond simply rendering using a new API standard, this project is also taking advantage of the new compute shaders available on WebGPU to calculate each cells' activation in parallel with each other.


Note: WebGPU is not supported on Safari browsers as of this writing, and needs to be explicitly enabled on firefox. Please refer to the logs below for details.
